# CaptionNest UI 设计规范

## 选定方向

| 层级 | 决策 |
|---|---|
| Direction | Launch Centered Hero Story（`launch-centered-hero-story`） |
| 原始 Theme | Acid Theme（`theme-ca63703dc9b7`，浅色） |
| 项目适配 | 经用户确认，保留方向结构，将 Acid 的荧光粉配色映射为 CaptionNest 品牌色 |
| 确认日期 | 2026-07-15 |

上游参考仅用于结构、节奏和表现语言，不复制其品牌、Logo、截图、专有布局或生产资产。

## 项目目标

CaptionNest 是面向 Windows 的本地优先双语字幕工具。首屏必须让用户立即理解并完成这条主流程：

```text
选择视频 → 确认识别与翻译设置 → 开始处理 → 查看阶段和日志 → 打开同目录 SRT
```

目标用户包括视频创作者、课程学习者和本地 AI 用户。界面默认使用简体中文，并保持可信、安静、精致。

## 首屏结构

继续使用所选 Direction 的 `centered-hero` 布局与 `narrative-proof` 内容模式：

1. 顶部只保留品牌、本地服务和算力状态，不放干扰主流程的导航。
2. 居中开场语直接说明结果：把视频放进来，带走一份双语字幕。
3. 开场语下方只设置一个主要证明面板，左侧是视频选择，右侧是任务摘要、三阶段进度和开始按钮。
4. 高级识别设置、Provider 凭据和模型管理采用渐进展开，不在首次进入时争夺注意力。
5. 任务启动后，主要证明面板原位切换为实时进度、日志与结果，不跳转到另一套页面结构。

首屏草图：`.ui-style-director/first-viewport-draft.svg`

## 信息层级

| 层级 | 内容 |
|---|---|
| Primary | 选择视频、开始生成字幕 |
| Secondary | 目标语言、识别模型、翻译服务、三阶段进度 |
| Supporting | 本地环境、CUDA、模型下载、隐私说明、实时日志 |

## 品牌色适配

原始 Acid Theme 的浅色外观、高对比强调和具有记忆点的几何细节继续保留；用户已授权用项目品牌色替换原主题的荧光粉。

| 角色 | 色值 | 用途 |
|---|---|---|
| Canvas | `#F7FBFA` | 应用背景 |
| Surface | `#FFFFFF` | 主要任务面板和设置卡片 |
| Surface Alt | `#FFF1D2` | 本地处理、隐私与结果提示 |
| Text | `#032F3B` | 标题和高优先级正文 |
| Muted | `#5F7379` | 次级说明与路径信息 |
| Accent | `#0B8492` | 可读按钮、焦点和当前状态 |
| Accent Bright | `#08D4ED` | 少量装饰、高亮轨道和状态光点 |
| Border | `#CFE2E3` | 面板边界和分隔线 |

亮青色不能作为白色小字号文字的底色；承载文字的主按钮使用 `#0B8492`。

## 排版与空间

- 使用系统无衬线字体栈：`Inter`、`Segoe UI`、`Microsoft YaHei`、sans-serif。
- 标题允许更强的尺寸对比，但正文保持安静、紧凑和易扫读。
- 首屏保持宽松分组，每个视觉区只承担一个决策。
- 桌面端主要证明面板最大宽度约 `1120px`；窄屏时改为单列，主要操作仍位于内容流末端。

## 组件与交互

- 继续使用现有 React 组件边界，必要时拆分小型专一组件，不引入整套营销组件库。
- Magic UI 仅作为动效语言参考，不作为必须安装的依赖。
- 动效只用于视频进入、阶段切换、进度变化和完成反馈，并遵守 `prefers-reduced-motion`。
- 设置使用可访问的原生表单语义；键盘焦点清晰，不用颜色作为唯一状态线索。
- API Key 只存在于当前任务内存，不出现在日志、草图或持久化文件中。

## 风险控制

| 风险 | 约束 |
|---|---|
| 方向偏营销页 | 所有文案服务于实际任务，不增加注册、宣传或虚构指标 |
| 留白隐藏必要设置 | 首屏显示任务摘要，高级配置可展开且状态始终可见 |
| 表现性动效干扰 | 只在状态变化时使用短动效，禁止持续漂浮和无意义循环 |
| 品牌亮青可读性不足 | 亮青仅用于图形高亮，交互主色使用深一些的 UI 主色 |

## 实施与验收契约

- 用户确认本文件和首屏草图后，方可修改 UI 代码。
- 实现必须保留居中开场语、单一主要证明面板、宽松密度和渐进设置模型。
- 时间轴仍由程序持有，模型只翻译稳定 ID 对应文本；视觉重构不得改变业务边界。
- 完成后运行 Python 测试、前端 lint/build，并用真实浏览器验证选择视频、设置、启动、进度和结果核心流程。
- 浏览器截图必须与本文件和已确认草图对照；有意偏差需记录原因。
